# NextAuth + Admin Guard Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **DO NOT COMMIT.** Per user preference, no `git commit` is run by implementers or controllers. All changes stay in the working tree; the user decides when and how to commit.

**Goal:** GitHub OAuth로 로그인 가능하게 하고, `ADMIN_GITHUB_ID`에 매칭되는 한 사람만 세션에 `isAdmin: true`를 부여한다. 미들웨어로 `/admin/**`를 가드하고, `/signin` 페이지와 `/admin` placeholder, 헤더 sign-in/out UI를 추가한다.

**Architecture:** NextAuth v5(Auth.js)를 사용. Edge 호환을 위해 config을 `auth.config.ts`(edge-safe)와 `auth.ts`(Node-only, Prisma adapter)로 분리. Prisma adapter + database 세션 전략으로 토큰을 DB에 저장(즉시 revoke 가능). `signIn` 콜백에서 admin 외 계정을 거부. 미들웨어는 edge runtime에서 `authConfig`만 사용해 `/admin/**`을 보호.

**Tech Stack:** Next.js 16, NextAuth v5(`next-auth@beta` 또는 `next-auth@5+`), `@auth/prisma-adapter`, Prisma 7 + `@prisma/adapter-pg`, Supabase Postgres, TypeScript strict.

**Reference spec:** `docs/superpowers/specs/2026-05-22-nextauth-design.md`

### Task 5: `auth.config.ts` 작성 (edge-safe)

**Files:**
- Create: `src/lib/auth.config.ts`

- [ ] **Step 1: 파일 작성**

Create `src/lib/auth.config.ts` with exactly:

```ts
import type { NextAuthConfig } from "next-auth";

const ADMIN_GITHUB_ID = process.env.ADMIN_GITHUB_ID;

export const authConfig = {
  pages: { signIn: "/signin" },
  callbacks: {
    async signIn({ account }) {
      if (account?.provider !== "github") return false;
      return account.providerAccountId === ADMIN_GITHUB_ID;
    },
    async session({ session, user }) {
      if (session.user) {
        session.user.id = user.id;
        session.user.isAdmin = true;
      }
      return session;
    },
    authorized({ auth, request }) {
      const isAdmin = !!auth?.user?.isAdmin;
      const isAdminPath = request.nextUrl.pathname.startsWith("/admin");
      if (isAdminPath) return isAdmin;
      return true;
    },
  },
  providers: [],
} satisfies NextAuthConfig;
```

```bash
pnpm typecheck
```

Expected: 에러 없음.

만약 `Session.user.id` 또는 `isAdmin`에 대한 타입 에러가 나면 Task 4의 declaration 파일이 잘 인식되는지 확인. 인식되지 않으면 `tsconfig.json` `include` 점검.

- [ ] **Step 4: §10.4 admin이 아닌 GitHub 계정 로그인 시도 → AccessDenied**

수동 또는 Playwright:
1. 브라우저로 `http://localhost:3000/signin` 접속
2. "GitHub으로 로그인" 클릭
3. **본인의 admin GitHub 계정이 아닌 다른 계정**으로 GitHub OAuth 동의 (예: 친구 계정, 부계정, 또는 새로 만든 sub-account)
4. 결과 URL이 `http://localhost:3000/signin?error=AccessDenied`로 회귀하는지 확인
5. 페이지에 "이 계정은 관리자가 아닙니다." 메시지 표시되는지

⚠️ 비-admin 계정이 없다면 이 검증은 스킵 가능. 대안: `.env.local`의 `ADMIN_GITHUB_ID`를 일시적으로 잘못된 값(예: `0`)으로 바꿔서 본인 계정을 비-admin으로 만들어 테스트 후 원복.

Expected: AccessDenied 회귀 + 에러 메시지 표시.
